## Overview

`HttpApi` provides a schema-first approach to building HTTP APIs. Define your
API once with `effect/Schema`, and get:

- Type-safe handlers and clients
- Automatic request/response validation
- Generated OpenAPI documentation
- End-to-end type safety

## Defining an API

An API is built up by chaining `.add(...)` calls — endpoints onto groups, and
groups onto the API. Endpoint constructors (`get`, `post`, …) take a name, a
path, and an options object containing the schemas.

```typescript
import {
  HttpApi,
  HttpApiEndpoint,
  HttpApiGroup
} from "effect/unstable/httpapi"
import { Schema } from "effect"

class User extends Schema.Class<User>("User")({
  id: Schema.Number,
  name: Schema.String,
  email: Schema.String
}) {}

class UsersApi extends HttpApiGroup.make("users")
  .add(
    HttpApiEndpoint.get("list", "/users", {
      success: Schema.Array(User)
    })
  )
  .add(
    HttpApiEndpoint.get("getById", "/users/:id", {
      success: User,
      params: Schema.Struct({ id: Schema.NumberFromString })
    })
  )
  .add(
    HttpApiEndpoint.post("create", "/users", {
      success: User,
      payload: Schema.Struct({
        name: Schema.String,
        email: Schema.String
      })
    })
  ) {}

class SystemApi extends HttpApiGroup.make("system")
  .add(HttpApiEndpoint.get("health", "/health")) {}

class Api extends HttpApi.make("api")
  .add(UsersApi)
  .add(SystemApi) {}
```

Endpoint options:

- `params` — schema for path parameters captured by `:name` segments
- `query` — schema for the URL query string
- `headers` — schema for required headers
- `payload` — schema for the request body (only on methods that have one)
- `success` — schema for the response body (defaults to no content)
- `error` — schema (or array of schemas) for typed errors

## Implementing handlers

`HttpApiBuilder.group(api, name, build)` registers handlers for one group. The
`build` callback receives a `handlers` value and returns it (or an effect that
returns it) after `.handle("endpointName", impl)` calls.

```typescript
import { Effect } from "effect"
import { HttpApiBuilder } from "effect/unstable/httpapi"

const UsersApiHandlers = HttpApiBuilder.group(
  Api,
  "users",
  (handlers) =>
    handlers
      .handle("list", () =>
        Effect.succeed([{ id: 1, name: "Alice", email: "alice@example.com" }])
      )
      .handle("getById", ({ path }) =>
        Effect.succeed({
          id: path.id,
          name: "Alice",
          email: "alice@example.com"
        })
      )
      .handle("create", ({ payload }) =>
        Effect.succeed({ id: 1, name: payload.name, email: payload.email })
      )
)

const SystemApiHandlers = HttpApiBuilder.group(
  Api,
  "system",
  (handlers) => handlers.handle("health", () => Effect.void)
)
```

Each handler receives a single context object exposing the decoded
`payload`, `path`, `urlParams`, `headers`, and the underlying `request`.

If you need to set up services before defining handlers, return an effect:

```typescript
const UsersApiHandlers = HttpApiBuilder.group(
  Api,
  "users",
  (handlers) =>
    Effect.gen(function*() {
      const users = yield* UserService
      return handlers
        .handle("list", () => users.list())
        .handle("getById", ({ path }) => users.getById(path.id))
        .handle("create", ({ payload }) => users.create(payload))
    })
)
```

## Serving the API

`HttpApiBuilder.layer(api)` builds the routes; provide the group handler layers
and pass the result to `HttpRouter.serve`.

```typescript
import { NodeHttpServer, NodeRuntime } from "@effect/platform-node"
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"
import { HttpApiBuilder } from "effect/unstable/httpapi"
import { createServer } from "node:http"

const ApiLive = HttpApiBuilder.layer(Api, {
  openapiPath: "/openapi.json"
}).pipe(
  Layer.provide([UsersApiHandlers, SystemApiHandlers])
)

const ServerLive = HttpRouter.serve(ApiLive).pipe(
  Layer.provide(NodeHttpServer.layer(createServer, { port: 3000 }))
)

Layer.launch(ServerLive).pipe(NodeRuntime.runMain)
```

`HttpApiBuilder.layer` only accepts `{ openapiPath }`. Metadata like the API
title and version is attached as annotations — see
[OpenAPI documentation](#openapi-documentation) below.

## Using the typed client

`HttpApiClient.make(Api)` produces a client shaped like the API:
`client.<group>.<endpoint>(args)`. It requires an `HttpClient` in the
environment.

```typescript
import { Effect, Layer } from "effect"
import { FetchHttpClient } from "effect/unstable/http"
import { HttpApiClient } from "effect/unstable/httpapi"

const program = Effect.gen(function*() {
  const client = yield* HttpApiClient.make(Api, {
    baseUrl: "http://localhost:3000"
  })

  const users = yield* client.users.list()
  const user = yield* client.users.getById({ path: { id: 123 } })
  const newUser = yield* client.users.create({
    payload: { name: "Alice", email: "alice@example.com" }
  })

  return { users, user, newUser }
}).pipe(Effect.provide(FetchHttpClient.layer))
```

`transformClient` lets you mutate the underlying `HttpClient` — for example, to
add retries:

```typescript
import { Schedule } from "effect"
import { HttpClient } from "effect/unstable/http"

const program = HttpApiClient.make(Api, {
  baseUrl: "http://localhost:3000",
  transformClient: (client) =>
    HttpClient.retryTransient(client, {
      schedule: Schedule.exponential("100 millis"),
      times: 3
    })
})
```

## OpenAPI documentation

### Serving docs with Scalar

`HttpApiScalar.layer` mounts a Scalar reference page. The OpenAPI spec is
generated from the API and served at `openapiPath` (set on
`HttpApiBuilder.layer`).

```typescript
import { Layer } from "effect"
import { HttpApiScalar } from "effect/unstable/httpapi"

const DocsLive = HttpApiScalar.layer(Api, { path: "/docs" })

const AppLive = Layer.mergeAll(ApiLive, DocsLive)
```

Visit `/docs` to see interactive API documentation.

### Customizing OpenAPI output

Attach metadata to the API via `OpenApi.annotations` and `.annotateMerge`:

```typescript
import { OpenApi } from "effect/unstable/httpapi"

class Api extends HttpApi.make("api")
  .add(UsersApi)
  .add(SystemApi)
  .annotateMerge(OpenApi.annotations({
    title: "My API",
    version: "1.0.0",
    description: "A type-safe API built with Effect"
  })) {}
```

The same pattern works on `HttpApiGroup` and individual endpoints to set
descriptions, summaries, tags, or servers per group/endpoint.

## Middleware

### Defining middleware

Middleware is a `Context.Service` declared via `HttpApiMiddleware.Service`. The
type parameters declare what the middleware requires and provides; the options
object declares its error and (optionally) security schemes.

```typescript
import { Context, Schema } from "effect"
import { HttpApiMiddleware } from "effect/unstable/httpapi"

class CurrentUser extends Context.Service<CurrentUser, { id: string }>()(
  "app/CurrentUser"
) {}

class Unauthorized extends Schema.TaggedErrorClass<Unauthorized>()(
  "Unauthorized",
  {}
) {}

class Authorization extends HttpApiMiddleware.Service<Authorization, {
  provides: CurrentUser
}>()("app/Authorization", {
  error: Unauthorized
}) {}
```

### Applying to groups or endpoints

Both `HttpApiGroup` and `HttpApiEndpoint` expose a `.middleware(M)` method:

```typescript
class UsersApi extends HttpApiGroup.make("users")
  .add(
    HttpApiEndpoint.get("profile", "/profile", { success: User })
  )
  .middleware(Authorization) {}
```

### Implementing middleware

Provide a `Layer.succeed(M, impl)` where `impl` matches the middleware's call
signature. For a plain middleware, that's a function from the handler effect to
a new handler effect.

```typescript
import { Effect, Layer } from "effect"

const AuthorizationLive = Layer.succeed(
  Authorization,
  (effect) =>
    Effect.provideService(effect, CurrentUser, { id: "user-123" })
)
```

For security-scheme middleware, declare `security` on the service and provide
an implementation per scheme. The implementation receives the decoded
credential.

```typescript
import { Redacted } from "effect"
import { HttpApiSecurity } from "effect/unstable/httpapi"

class ApiKeyAuth extends HttpApiMiddleware.Service<ApiKeyAuth, {
  provides: CurrentUser
}>()("app/ApiKeyAuth", {
  security: {
    apiKey: HttpApiSecurity.apiKey({ in: "header", key: "x-api-key" })
  }
}) {}

const ApiKeyAuthLive = Layer.succeed(ApiKeyAuth)({
  apiKey: (effect, { credential }) =>
    Effect.provideService(effect, CurrentUser, {
      id: Redacted.value(credential)
    })
})
```

### Client-side middleware

`HttpApiMiddleware.layerClient` wires middleware logic into the generated
client — for example, to inject an auth header:

```typescript
import { HttpClientRequest } from "effect/unstable/http"
import { HttpApiMiddleware } from "effect/unstable/httpapi"

const AuthorizationClient = HttpApiMiddleware.layerClient(
  Authorization,
  ({ next, request }) =>
    next(HttpClientRequest.bearerToken(request, "my-token"))
)
```

## Security schemes

`HttpApiSecurity` describes how an endpoint or group authenticates requests.
The module exposes constructors for the standard schemes — `bearer`, `basic`,
and `apiKey` — plus `http` for custom HTTP authentication schemes.

```typescript
import { HttpApiSecurity } from "effect/unstable/httpapi"

// Standard Bearer token: Authorization: Bearer <token>
const Bearer = HttpApiSecurity.bearer

// Custom HTTP scheme: Authorization: DPoP <token>
const DPoP = HttpApiSecurity.http({ scheme: "DPoP" })

// API key via header, query parameter, or cookie
const ApiKey = HttpApiSecurity.apiKey({
  in: "header",
  key: "x-api-key"
})
```

Attach a security scheme to a middleware (as shown above) and the credential
will be decoded automatically before the handler runs. Custom HTTP schemes
defined with `HttpApiSecurity.http` are also rendered into the generated
OpenAPI document.

## Complete example

```typescript
import { NodeHttpServer, NodeRuntime } from "@effect/platform-node"
import { Effect, Layer, Schema } from "effect"
import { HttpRouter } from "effect/unstable/http"
import {
  HttpApi,
  HttpApiBuilder,
  HttpApiEndpoint,
  HttpApiGroup,
  HttpApiScalar,
  OpenApi
} from "effect/unstable/httpapi"
import { createServer } from "node:http"

class User extends Schema.Class<User>("User")({
  id: Schema.Number,
  name: Schema.String
}) {}

class UsersApi extends HttpApiGroup.make("users")
  .add(
    HttpApiEndpoint.get("list", "/users", {
      success: Schema.Array(User)
    })
  )
  .add(
    HttpApiEndpoint.get("getById", "/users/:id", {
      success: User,
      params: Schema.Struct({ id: Schema.NumberFromString })
    })
  ) {}

class Api extends HttpApi.make("api")
  .add(UsersApi)
  .annotateMerge(OpenApi.annotations({
    title: "Users API",
    version: "1.0.0"
  })) {}

const UsersApiHandlers = HttpApiBuilder.group(
  Api,
  "users",
  (handlers) =>
    handlers
      .handle("list", () =>
        Effect.succeed([{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }])
      )
      .handle("getById", ({ path }) =>
        Effect.succeed({ id: path.id, name: "Alice" })
      )
)

const ApiLive = HttpApiBuilder.layer(Api, {
  openapiPath: "/openapi.json"
}).pipe(Layer.provide(UsersApiHandlers))

const DocsLive = HttpApiScalar.layer(Api, { path: "/docs" })

const ServerLive = HttpRouter.serve(
  Layer.mergeAll(ApiLive, DocsLive)
).pipe(
  Layer.provide(NodeHttpServer.layer(createServer, { port: 3000 }))
)

Layer.launch(ServerLive).pipe(NodeRuntime.runMain)
```
