Effect is a TypeScript library for building robust, type-safe applications with powerful effect management, error handling, and resource management capabilities.

## Core Modules

The core Effect library provides the fundamental building blocks for effectful programming.

### Effect

The `Effect` type represents a computation that may succeed with a value of type `A`, fail with an error of type `E`, or require services of type `R`.

```ts
type Effect<A, E = never, R = never>
```

**Key Operations:**
- `Effect.succeed` ‚Äî Create a successful effect
- `Effect.fail` ‚Äî Create a failed effect
- `Effect.gen` ‚Äî Generator-based effect composition
- `Effect.flatMap` ‚Äî Sequential composition
- `Effect.map` ‚Äî Transform success values
- `Effect.catch` ‚Äî Error recovery
- `Effect.provide` ‚Äî Provide service implementations

[Learn more about Effect basics ‚Üí](/core-concepts/effect-basics)

### Schema

Define data shapes, validate unknown input, and transform values between formats.

**Key Features:**
- Elementary schemas (primitives, literals, strings, numbers)
- Composite schemas (structs, tuples, arrays, records, unions)
- Validation rules with `.check()`
- Transformations with `decodeTo` and `encodeTo`
- Opaque types and classes
- JSON Schema generation

[Learn more about Schema ‚Üí](/core-modules/schema)

### Stream

Effectful streams for processing sequences of values with backpressure support.

**Key Operations:**
- `Stream.make` ‚Äî Create a stream from values
- `Stream.fromIterable` ‚Äî Create from an iterable
- `Stream.map` ‚Äî Transform elements
- `Stream.filter` ‚Äî Filter elements
- `Stream.flatMap` ‚Äî Flatten nested streams
- `Stream.runCollect` ‚Äî Collect all elements
- `Stream.run` ‚Äî Run stream with a sink

[Learn more about Stream ‚Üí](/core-modules/stream)

### Layer

Manage service dependencies and their lifecycles.

**Key Operations:**
- `Layer.succeed` ‚Äî Create a layer from a value
- `Layer.effect` ‚Äî Create a layer from an effect
- `Layer.scoped` ‚Äî Create a scoped layer with cleanup
- `Layer.provide` ‚Äî Wire layer dependencies
- `Layer.merge` ‚Äî Combine layers

[Learn more about Layer ‚Üí](/core-modules/layer)

## Data Structures

Effect provides immutable, persistent data structures optimized for functional programming.

### Collection Types

- **`Chunk`** ‚Äî Immutable array-like collection with O(1) append
- **`HashMap`** ‚Äî Persistent hash map with structural equality
- **`HashSet`** ‚Äî Persistent hash set with structural equality
- **`List`** ‚Äî Linked list with efficient head operations
- **`RedBlackTree`** ‚Äî Ordered map with O(log n) operations
- **`SortedSet`** ‚Äî Ordered set with O(log n) operations

### Concurrent Primitives

- **`Queue`** ‚Äî Concurrent queue with backpressure
- **`PubSub`** ‚Äî Publish-subscribe messaging
- **`Ref`** ‚Äî Concurrent mutable reference
- **`Deferred`** ‚Äî One-time settable promise
- **`Semaphore`** ‚Äî Resource limiting and coordination
- **`Mutex`** ‚Äî Mutual exclusion lock

[Learn more about data structures ‚Üí](/core-modules/data-structures)

## Configuration and Resources

### Config

Type-safe configuration management with environment variable support.

**Key Operations:**
- `Config.string` ‚Äî String configuration
- `Config.number` ‚Äî Number configuration
- `Config.boolean` ‚Äî Boolean configuration
- `Config.array` ‚Äî Array configuration
- `Config.nested` ‚Äî Nested configuration
- `Config.withDefault` ‚Äî Default values

[Learn more about Config ‚Üí](/core-modules/config)

### Cache

Memoization and caching with TTL support.

**Key Features:**
- Time-to-live (TTL) expiration
- Size-based eviction
- Effectful lookups
- Concurrent access coordination

[Learn more about Cache ‚Üí](/core-modules/cache)

## Scheduling and Timing

### Schedule

Define retry and repeat policies with backoff strategies.

**Key Schedules:**
- `Schedule.forever` ‚Äî Repeat indefinitely
- `Schedule.recurs` ‚Äî Repeat n times
- `Schedule.exponential` ‚Äî Exponential backoff
- `Schedule.spaced` ‚Äî Fixed delay between runs
- `Schedule.jittered` ‚Äî Add randomness to delays

[Learn more about Schedule ‚Üí](/core-modules/schedule)
